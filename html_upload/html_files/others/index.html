Methods

Method Description Parameters Default Values
getCurrentPosition(success, error?, options?) Gets the device’s current location once and invokes success with a GeolocationResponse.
success(position) — required
error(err) — optional
options — optional (GeolocationOptions)
 Uses GeolocationOptions defaults.

GeolocationOptions (for getCurrentPosition / watchPosition)

Option Type Default Description
enableHighAccuracy boolean false Use GPS/high-accuracy mode (higher battery usage).
timeout number (ms) Infinity Max time to wait before failing with a timeout error.
maximumAge number (ms) 0 Accept a cached location up to this age.
distanceFilter iOS only number (meters) 0 Minimum movement (in meters) before an update event is emitted.

On Android, calls may resolve immediately if a recent cached location is available, or take longer while a fresh fix is obtained.

GeolocationConfiguration

Key Type Default Description
skipPermissionRequests boolean false If true, you must request permissions yourself.
authorizationLevel iOS only 'whenInUse' | 'always' 'whenInUse' Permission scope to request from the user.

{

    altitude?: number,
    accuracy: number,
    altitudeAccuracy?: number,
    heading?: number,

}

GeolocationError

Key Type Description
code 1 | 2 | 3
1 — Permission denied
2 — Position unavailable
3 — Timeout
message string Human-readable description of the error.

Usage Tips

  • Prefer watchPosition for live tracking; always call clearWatch when finished.
  • On iOS, tune distanceFilter to reduce battery drain.
  • Handle permission flows differently on iOS and Android when using skipPermissionRequests.
